## 存储结构

### 顺序存储

按**满二叉树**的结点层次编号，依次存放在二叉树中的数据元素。需要注意的是编号顺序为从上往下从左往右，如下：

```c
// 二叉树顺序存储表示
#define MAXTSIZE 100
Typedef TElemType SqBiTree[MAXTSIZE]
SqBiTree bt
```

顺序存储缺点：

- 长度固定
- （子）单支树多的情况下，需要很多空间来存储占位的0元素

顺序存储比较适合满二叉树和完全二叉树（从根往下数，除了最下层外都是全满（都有两个子节点），而最下层所有叶结点要么全满要么向左。）

### 链式存储

存储方式: (lchild(指向左孩子结点), data, rchild(指向右孩子结点))

**比较适合经常找后继的情况**

二叉树存储结构：

```c
typedef struct BiNode{
  TElemType data;
  struct BiNode *lchild *rchild
}BiNode, *BiTree
```

练习：在n个结点的二叉链表中有多少个空指针域？

分析：必有2n个指针；除根结点外每个结点有且仅有一个父节点，这条代表关系的连线会占用一个指针，所以有n-1个被占用的指针，那么空指针数量为 2n - (n - 1)

**比较适合经常找前驱的情况如下：**

三叉链表：存储方式为：(lchild(指向左孩子结点), data, parent(指向父结点), rchild(指向右孩子结点))

存储结构如下：

```c
typedef struct TriTNode{
  TElemType data;
  struct TriTNode *lchild *parent *rchild
}TriTNode, *TriTree
```
